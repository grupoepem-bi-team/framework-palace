# Palace Framework — Roadmap por Etapas

## Objetivo
Convertir el sistema en una infraestructura confiable, segura y escalable.

## Etapas

### 1. Testing
- Crear estructura de tests
- Tests unitarios (context, pipeline, memory)
- Tests de integración
- Mock de Ollama

### 2. Sandbox
- Runner con Docker
- Ejecución aislada
- Validación previa

### 3. Observabilidad
- Logging estructurado
- Métricas por pipeline
- Trazabilidad completa

### 4. Agentes
- Definir roles (planner, dev, ops, reviewer)
- Prompts persistentes
- Flujo multi-agente

### 5. Memoria
- Scoring
- Deduplicación
- Expiración

### 6. Optimización
- Uso eficiente de modelos
- Reducción de costos
- Mejorar latencia
